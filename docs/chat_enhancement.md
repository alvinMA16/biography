# 增强模式干预系统方案

## 问题背景

当前干预系统存在两个核心问题：
1. **时序竞态**：干预在 ASR 结束(事件459)时触发 LLM 判断(500-2000ms)，但豆包已立即开始生成回复。等干预注入时，当轮回复已经生成完毕，干预实际只影响下一轮。
2. **干预内容弱**：注入格式、system_role 指令、prompt 输出都偏"建议"而非"指令"，豆包不会严格遵循。

## 干预类型分析

四种干预类型本质上分为两类：

### 行为指令型（用 510 指令注入）
| 类型 | 场景 | 510 注入内容 |
|------|------|-------------|
| **话题偏离** | AI 在具体事物上展开太久，没有转向"人"和"情" | 明确的转向指令，如"你下一句话要从做菜转到问谁教她做的" |
| **对话停滞** | 连续几轮在同一个点打转，没有新信息 | 换角度指令，如"换个角度，问下班后的生活" |
| **重要线索** | 用户提到值得深挖的人物/地点/事件 | 追问指令，如"用户提到了张老师，追问张老师是什么样的人" |

### 知识注入型（用 510 知识注入 + 预加载）
| 类型 | 场景 | 注入方式 |
|------|------|---------|
| **时代触发** | 用户提到与时代相关的内容（下乡、粮票等） | **预加载**：StartSession 时在 dialog_context 放精简版时代记忆；**按需注入**：触发时用 510 注入具体的相关时代背景知识（非指令） |

## 方案：预判断 + 即时注入

### 核心思路

在 TTS 结束（事件 359）后启动 LLM 判断，判断完成后**立即**通过 510 注入上下文（不等 459）。由于注入发生在用户沉默间隙，等用户说完话（459）模型开始生成时，指令/知识**早已在上下文中**，完全没有竞态。

```
359(TTS结束) → 启动LLM判断 → 判断完成 → 立即发510注入上下文
[用户思考、开始说话]
459(ASR结束) → 模型生成回复时，510的内容已经在上下文里了
```

### 注入格式

**行为指令型**（话题偏离/停滞/重要线索）：
```json
{
    "items": [
        {"role": "user", "text": "【系统指令 - 必须执行】你下一句话要把话题从讨论做菜细节转到问用户谁教她做的这道菜"},
        {"role": "assistant", "text": "收到指令，我的下一句回复将严格按此要求执行。"}
    ]
}
```

**知识注入型**（时代触发）：
```json
{
    "items": [
        {"role": "user", "text": "【背景知识】关于用户提到的"下乡"：1968年上山下乡运动，大量城市知识青年被送往农村，很多人一去就是近十年。你可以用这些背景自然地和用户聊这段经历。"},
        {"role": "assistant", "text": "好的，我了解了这段历史背景。"}
    ]
}
```

### 时代记忆预加载

在 StartSession 建立连接时，将精简版时代记忆放入 `dialog_context`，让模型从一开始就有基础的历史背景知识。对话中触发时再通过 510 注入具体的相关片段。

## 前端调试信息

干预触发时，蓝色调试气泡显示三层信息：
1. **干预类型**：话题偏离 / 停滞 / 重要线索 / 时代触发
2. **注入机制**：510指令注入 / 510知识注入
3. **干预内容**：具体的指令或知识文本

## 涉及文件

| 文件 | 改动内容 |
|------|---------|
| `backend/app/api/realtime_enhanced.py` | 判断完成后立即注入510，不等459 |
| `backend/app/services/doubao_realtime_enhanced.py` | 区分指令注入和知识注入两种格式 |
| `backend/app/services/intervention_service.py` | 返回干预类型+机制+内容 |
| `backend/app/prompts/realtime_chat_enhanced.py` | 强化 system_role 指令遵循 |
| `backend/app/prompts/intervention_*.py` | 加入 ai_response，输出直接指令 |
| `backend/app/config.py` | timeout 调整 |
| `web/js/realtime-chat.js` | 气泡显示干预类型和机制 |

## 验证方式

1. 部署后打开 debug 模式观察蓝色气泡
2. 气泡应显示 `[话题偏离|510指令注入] 具体内容...` 格式
3. 核心验证：气泡弹出后，豆包的**下一句回复**是否跟随了干预方向
4. 观察后端日志确认判断在 TTS 结束后立即完成并注入
